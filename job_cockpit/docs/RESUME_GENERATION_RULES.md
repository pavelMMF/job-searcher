# Resume Generation Rules

These rules are the standing contract for every tailored resume generated by Job Cockpit.

Also follow `job_cockpit/docs/APPLICATION_TEXT_GENERATION_RULES.md` for anti-generic-AI style, cover letters, and application messages.

## Core Principle

The resume must be tailored, but truthful. It may reorder, emphasize, compress, and rewrite real experience for a vacancy. It must not invent employers, dates, degrees, tools, metrics, work authorization, citizenship, or projects.

## Resume Generation Pipeline

Every tailored resume must follow this sequence:

1. Parse the vacancy title, company, location, requirements, tools, seniority and domain signals.
2. Classify the role family: product/affiliate/data analytics, analytics engineering, or middle data science.
3. Load `master_resume.json` as the factual source of truth.
4. Read `experience_bank` from `master_resume.json` for additional real projects, tools and bullets not shown in the short base resume.
5. Use the original PDF resume text only as a style and completeness reference.
6. Build a deterministic fallback resume payload before calling LLM.
7. Ask LLM to rewrite only within the fallback payload structure.
8. Preserve every real role and every required period.
9. Reorder skills and bullets by vacancy relevance, but keep only real tools and real experience.
10. Validate forbidden phrases, forbidden metadata, required periods and minimum role coverage.
11. Render the PDF with the aurora header and golden/brown section style.
12. If validation or rendering fails, attach the standard resume instead of a risky tailored resume.

## Required Structure

Every generated PDF resume must include:

1. Aurora header image from the original resume style.
2. Name and headline.
3. Contact line with phone, email, preferred messenger, and location.
4. Summary.
5. Highlights.
6. Key skills.
7. Experience.
8. Education and training.

The resume must not include internal generation metadata such as:

- `Tailored for ...`
- `Generated ...`
- job-cockpit debug text
- agent notes
- hidden scoring or matching explanations

## Experience Rules

The `Experience` section must always include every real role from `job_cockpit/config/master_resume.json`, including older roles.

Current required roles:

- `Apr 2025 - Present - Data Analyst, iGaming`
- `Apr 2023 - Mar 2025 - Data Analyst, iGaming`

Each role must have at least 2 bullets when enough source bullets exist. The generator may add more bullets to the most relevant roles, but it must never drop the older `Apr 2023 - Mar 2025` role just because newer roles matched more keywords.

Experience bullets should be selected by relevance to the vacancy while preserving coverage:

- data/product/affiliate analytics roles should emphasize SQL, BI, dashboards, retention, cohorts, LTV, A/B testing, GGR, reporting, product metrics, marketing/product/finance stakeholders;
- data science roles should emphasize Python, SQL, churn, prediction, ML, RAG/vector search, AI agents, embeddings, knowledge graphs, model explanation, analytics automation;
- analytics engineering roles should emphasize SQL, ETL, data marts, API integrations, PostgreSQL, large datasets, Docker, reporting infrastructure.

## Style Guide

Tone:

- human, direct, confident;
- no overstuffed keyword soup;
- no marketing fluff;
- no exaggerated seniority;
- no fake claims.

Visual style:

- clean, readable, recruiter-friendly;
- warm dark text with golden/brown section headings;
- no blue section headers;
- no large decorative blocks except the aurora header;
- compact enough for fast 10-second screening;
- stable spacing, no overlapping text.

Text rules:

- section titles must be `Summary`, `Highlights`, `Key skills`, `Experience`, `Education and training`;
- cover-letter style prose must not leak into resume bullets;
- bullets should be concise and impact-oriented;
- use ASCII hyphen `-` for bullets and separators.
- avoid generic AI-like phrases such as `proven track record`, `results-oriented`, `dynamic`, `innovative`, `cutting-edge`, `excellent communicator`, `team player`, and unsupported `actionable insights`.

## Tailoring Rules

Tailoring may:

- choose the most vacancy-aligned truthful headline from `headline_variants`, such as `Product Data Analyst`, `Senior Marketing Analytics Engineer`, `Senior Analytics Engineer`, `Senior Data Analyst`, `Senior Product Analyst`, `Senior Marketing Data Analyst`, `Product Affiliate Analyst`, `BI Analyst`, or `Data Scientist`;
- rewrite the summary for the role/company;
- reorder skills by vacancy relevance;
- select the strongest real bullets per role;
- emphasize matching tools and domains already present in the master resume.
- use vacancy wording when it describes real or adjacent experience already present in the master resume.
- rewrite experience bullets more strongly toward the vacancy by foregrounding matching real responsibilities, tools, domains, stakeholders, metrics, infrastructure, dashboards, experiments, marketing/traffic/funnel work, AI automation, or analytics engineering.
- merge, compress, and paraphrase adjacent real bullets so the experience feels purpose-built for the role, while keeping the facts interview-defensible.

Tailoring may not:

- invent a company-specific project;
- claim experience with tools absent from the master resume;
- list a vacancy skill as the candidate's skill unless it is present in the master resume, reference resume, or explicit answer bank;
- claim relocation/work authorization that is not in profile;
- change dates or employers;
- change real experience role titles inside the `Experience` section;
- remove an entire real role;
- hide older experience.

## Experience Bank

Use `experience_bank` for real experience that does not fit cleanly into the short resume but can be used for tailoring.

Each item should include:

- `id`;
- `source`;
- `role_period`;
- `tags`;
- `skills`;
- `bullets`;
- optional `notes`.

The bank can include anonymized projects, extra responsibilities, tools used in a limited scope, domain-specific work, prototypes, analysis types, stakeholder work and automation tasks. It must still be interview-defensible.

To import a separate bank file:

```powershell
python job_cockpit\core\import_experience_bank.py job_cockpit\config\experience_bank.sample.json
```

## Fallback Rules

If a vacancy has too little information to tailor responsibly, use the standard resume path from `master_resume.json` rather than generating a misleading tailored resume.

If PDF generation fails, attach the standard resume and log the failure.

## Quality Checks

Before treating the generator as correct, verify at least one generated PDF text contains:

- `Apr 2025 - Present`
- `Apr 2023 - Mar 2025`

And does not contain:

- `Dec 2025 - Present`
- `Product Analyst | Retention Analyst | Data Scientist`
- `Tailored for`
- `Generated 20`
- forbidden AI-style buzzwords from `APPLICATION_TEXT_GENERATION_RULES.md`
